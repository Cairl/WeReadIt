## WEREAD_ANDROID_TOKEN / WEREAD_IOS_TOKEN 过期报错修复

**问题**：当 WEREAD_ANDROID_TOKEN 或 WEREAD_IOS_TOKEN 过期（errcode=-2012）时，ExchangeError 被 app.py 的 `except Exception` 兜底捕获，转成字符串拼接到成功推送末尾，exit_code 仍为 0。用户无法从 GitHub Actions 状态或推送标题中感知 Token 已过期。

**修复内容**：

1. `src/wereadit/app.py`：
   - 新增导入 `ExchangeError` 和 `ERRCODE_TOKEN_EXPIRED`
   - 在兑换 try/except 中区分 `ExchangeError(errcode==-2012)` 和其他异常
   - Token 过期时：设置 `exit_code = 1`，exchange_summary 包含"请重新抓包更新 Secret 中的 Token"提示

2. `src/wereadit/core/exchanger.py`：
   - 查询奖励的 `_call_exchange` 调用加上 try/except，与单个奖励兑换循环的 Token 过期 re-raise 保持一致

**行为变化**：
- Token 过期 → GitHub Actions 退出码 1（失败）→ 推送消息中明确提示"请重新抓包更新 Secret"
- 其他兑换错误 → 行为不变，错误信息拼接到推送末尾，exit_code=0
